<Part id="oop-part-data-types-table">

Java cung cấp 8 kiểu dữ liệu nguyên bản, dưới đây là các kiểu dữ liệu dùng để xử lý số học:

| Kiểu dữ liệu (Type Name) | Kích thước (Bytes) | Phạm vi giá trị (Range) |
| --- | --- | --- |
| byte | 1 | -128 đến 127 ($-2^7$ đến $2^7 - 1$) |
| short | 2 | -32,768 đến 32,767 ($-2^{15}$ đến $2^{15} - 1$) |
| int | 4 | $-2^{31}$ đến $2^{31} - 1$ (Kiểu số nguyên mặc định) |
| long | 8 | $-2^{63}$ đến $2^{63} - 1$ |
| float | 4 | Âm: -3.4028235E+38 đến -1.4E-45 | Dương: 1.4E-45 đến 3.4028235E+38 |
| double | 8 | Âm: -1.7976931E+308 đến -4.9E-324 | Dương: 4.9E-324 đến 1.7976931E+308 (Mặc định) |


:::highlight
<b>Quy ước thực hành:</b> Trừ khi có ràng buộc bài toán cụ thể về tối ưu bộ nhớ hoặc dải giá trị rất lớn, lập trình viên luôn chọn <code>int</code> cho số nguyên và <code>double</code> cho số thực.
:::

</Part>

<Part id="oop-part-operators-precedence">

Quá trình đánh giá biểu thức số học phức tạp dựa trên quy tắc ưu tiên (Precedence) từ cao đến thấp và hướng kết hợp (Associativity):

1. Dấu ngoặc <code>()</code> - Hướng kết hợp: Trái sang Phải.
2. Toán tử tăng/giảm hậu tố <code>++</code>, <code>--</code> - Hướng kết hợp: Phải sang Trái.
3. Toán tử tăng/giảm tiền tố <code>++</code>, <code>--</code> và toán tử đơn tử <code>+</code>, <code>-</code> - Hướng kết hợp: Phải sang Trái.
4. Phép toán nhân, chia, lấy số dư <code>*</code>, <code>/</code>, <code>%</code> - Hướng kết hợp: Trái sang Phải.
5. Phép toán cộng, trừ <code>+</code>, <code>-</code> - Hướng kết hợp: Trái sang Phải.
6. Toán tử gán và toán tử thu gọn <code>=</code>, <code>+=</code>, <code>-=</code>, <code>*=</code>, <code>/=</code>, <code>%=</code> - Hướng kết hợp: Phải sang Trái.

</Part>

<Part id="oop-part-casting-playground">

Lập trình viên Java thường gặp lỗi logic kinh điển khi thực hiện phép chia giữa hai số nguyên. Hãy xem xét đoạn mã lỗi sau:

```java
double d;
int i = 31415;
d = i / 10000; // Sai lầm: i / 10000 thực chất là phép chia 2 số nguyên, trả về kết quả nguyên là 3. Sau đó số 3 mới được chuyển thành 3.0 gán cho d.
```

<b>Giải pháp sửa lỗi:</b> Thực hiện ép kiểu tường minh hoặc thay đổi toán hạng thành số thực literal: <code>d = (double)i / 10000;</code> hoặc <code>d = i / 10000.0;</code>.

:::definition
<b>Ép kiểu tường minh (Type casting):</b> Cú pháp: <code>(datatype) value;</code>. Ví dụ: Nếu <code>d = 3.987;</code>, câu lệnh <code>i = (int) d;</code> sẽ cắt bỏ hoàn toàn phần thập phân và gán giá trị nguyên là 3 cho biến i.
:::

<Part id="oop-part-temp-conversion">

Chương trình thực hiện chuyển đổi giá trị nhiệt độ và in kết quả ra màn hình:

```java
public class Temperature {
    public static void main(String[] args) {
        double fahrenheit, celsius;
        fahrenheit = 123.5;
        celsius = (5.0 / 9) * (fahrenheit - 32);
        System.out.println("Celsius: " + celsius);
    }
}
```

:::highlight
<b>Ghi chú kỹ thuật quan trọng:</b> Biểu thức <code>5.0/9</code> là bắt buộc để kích hoạt cơ chế tính toán số thực. Nếu viết <code>5/9</code>, Java hiểu là chia hai số nguyên và trả về kết quả bằng 0, khiến toàn bộ công thức bị sai. Ký tự <code>+</code> trong câu lệnh in đóng vai trò là toán tử nối chuỗi (Concatenate operator), tự động ép kiểu các giá trị số sang chuỗi văn bản.
:::

</Part>
